  EFFECT COMMAND iNFORMATiON

 

 Every effect command (as shown below) consists of two

 separated bytes. The first is effect byte and the second is info byte.

 The first byte (effect_def) is fixed, and carries the information

 that points to second "interactive" byte (effect).

 If the second byte is supposed to be partially fixed, then the info

 byte carries additional information that points to last 4 bits

 of info byte that are "interactive". If both effect and info bytes

 are fixed, then such effect is called an "extended command".

 /┤DLiB TR/┤CK3R ][ uses total count of 86 effects, and 16 extended

  FiLE FORMAT iNFORMATiON

 

 All data files that are in use of /┤DLiB TR/┤CK3R ][ are roughly

 composed of 2 parts. The first part (header) contains file authenticity

 information like ID string, CRC, format version, and is ended

 with one or more values signifying the length of next (compressed) part.

 You can distinguish the type of compression algorithm (FFver 1-8)

 from file format version: 1/5=SixPack, 2/6=LZW, 3/7=LZSS and 4/8=none.

 Since /┤DLiB TR/┤CK3R ][ revision 31, all data files
 A2M,A2T,A2P,A2i,A2B,A2W,A2F) are compressed using Jibz's aPLib
 compression library.
 Let's take a look on each of these file types.
